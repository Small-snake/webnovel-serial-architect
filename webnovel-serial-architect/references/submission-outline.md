# Submission Outline

Use this reference when the user needs a pitch-ready or editor-facing outline.

## Principle

A submission outline should prove market promise, readable opening, sustainable conflict, and a controllable long-form direction. It does not need every later chapter fixed.

Mark each item as:

- **Fixed**: needed for the premise or opening.
- **Tentative**: useful direction, can evolve after writing.
- **Deferred**: not needed for submission or opening yet.

## Build Order

1. One-sentence selling point.
2. Genre and target reader.
3. Core emotional promise.
4. Protagonist identity, desire, pressure, and advantage.
5. Core conflict and antagonistic force.
6. Payoff mechanism and repeatable爽点.
7. First ten chapters hook chain.
8. First arc mainline.
9. Medium-to-long escalation path.
10. Important characters and relationship tensions.
11. Ending direction, flexible unless required.
12. 200-300 character copy.

## Submission Outline Shape

Use this structure:

```markdown
# Submission Outline

## Title Directions

## Genre And Target Reader

## One-Sentence Selling Point

## Core Promise

## Protagonist

## Advantage / Golden Finger / Core Method

## Core Conflict

## Repeatable Payoff Mechanism

## First Ten Chapters

## First Arc

## Long-Form Escalation

## Key Characters And Relationships

## Ending Direction

## 200-300 Character Copy

## Open Questions
```

## Quality Check

Before presenting the outline, check:

- Does the opening conflict appear within the first chapter?
- Does the protagonist have an active goal?
- Is the promise understandable without long explanation?
- Can the payoff repeat without becoming mechanical?
- Does the first arc produce enough momentum for continuing chapters?
- Are deferred items clearly marked so the user can start writing?
