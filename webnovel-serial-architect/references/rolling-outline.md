# Rolling Outline

Use this reference when the user has already written chapters or wants to plan while serializing.

## Principle

Extract the current story state first. Do not overwrite the user's written material with a new full outline unless they ask for a rebuild.

## Intake From Existing Text

When the user provides chapters, summaries, or fragments, extract:

- Current protagonist goal.
- Current external conflict.
- Current internal pressure.
- Characters introduced.
- Relationship changes.
- Payoffs already delivered.
- Foreshadowing already planted.
- Unresolved questions.
- Timeline facts.
- Reader expectation at the current stopping point.

## Rolling Records

Maintain these as living records:

- **Inspiration bank**: usable fragments and deferred ideas.
- **Character state cards**: current desire, pressure, relationship, secret, change.
- **Foreshadowing log**: planted clue, expected payoff, deadline or possible arc.
- **Timeline**: chapter order, day/time, cause and effect, continuity constraints.
- **Next-chapter queue**: conflicts, reveals, payoffs, hooks available now.

## Next 5-10 Chapters

When extending the outline, provide:

```markdown
## Current State

## Available Story Lines

## Recommended Direction

## Next 5-10 Chapter Plan

## Needed Setup

## Deferred Decisions
```

## Continuity Rules

- Respect established facts unless the user asks for a retcon.
- Separate confirmed canon from tentative plans.
- Use existing unresolved pressure before adding new large systems.
- Introduce new characters only when they perform a story function.
- Keep deferred ideas visible so they can be reused later.
