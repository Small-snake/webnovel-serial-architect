# Opening Chapters

Use this reference when the user wants to start writing, design chapter hooks, or plan the first 3-10 chapters.

## Opening Principle

The opening should quickly establish protagonist, pressure, conflict, promise, and forward motion. Avoid front-loading full worldbuilding.

## First Chapter Jobs

The first chapter should usually do most of these:

- Place the protagonist under immediate pressure.
- Show what the protagonist wants or refuses.
- Reveal a contradiction, secret, advantage, or wound.
- Create a concrete conflict with stakes.
- Deliver one small payoff or sharp reversal.
- End with a hook that naturally pulls chapter two.

## Opening Pattern Options

Offer 2-3 options when the user only has a premise:

- **Crisis opening**: Start at the moment something goes wrong.
- **Humiliation opening**: Start with status pressure, then promise reversal.
- **Mission opening**: Start with a task that exposes the world and stakes.
- **Mystery opening**: Start with an impossible event or unanswered clue.
- **Relationship opening**: Start with emotional conflict, debt, betrayal, or reunion.
- **Power reveal opening**: Start with a constrained use of the protagonist's special method.

## First Ten Chapters

Plan chapters as a hook chain:

```markdown
## Chapter N

- Scene task:
- Protagonist goal:
- Conflict:
- Payoff:
- New question / hook:
- Setup for later:
```

## Rhythm Check

For the first ten chapters, check:

- Chapter 1 establishes pressure and hook.
- Chapters 2-3 escalate rather than explain.
- A first meaningful payoff arrives early.
- The protagonist makes choices, not only receives information.
- Setup is attached to action.
- Each chapter ending creates a next-step question.
